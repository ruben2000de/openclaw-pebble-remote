# Code of Conduct

This project uses a small, practical conduct rule:

Be kind, be specific, and do not make maintainers or users feel unsafe.

## Expected behavior

- Report bugs with enough detail to reproduce them.
- Criticize ideas and code, not people.
- Redact secrets and private data before posting logs or screenshots.
- Assume good faith, but accept maintainer decisions on scope and safety.

## Unacceptable behavior

- Harassment, threats, hate speech or personal attacks.
- Posting tokens, private endpoints or other people's data.
- Pressuring maintainers to add unsafe wrist-triggered actions.
- Deliberately bypassing documented security boundaries.

## Enforcement

Maintainers may close issues, edit/remove harmful content where platform tools allow it, or block users who repeatedly make the project unsafe or unmaintainable.

This deliberately lightweight code is meant to protect a small open-source project without importing unnecessary bureaucracy.
